## ️ℹ️  What is this project about?
<hr>

This project is a CLI tool that allows its user to break down a signal that is inputted by the user.
When it first starts up, it searches the current running directory for files that it can potentially process, if there are multiple,
it gets an input from the user for which file to use. There are also several flags that can be added to the command used to
run the program which allows other applications to better communicate with it and take its outputs.


### 📖 How is this achieved?

Math. Complex math, but literally, it uses complex numbers by shifting the signal into a "frequency" domain where an algorithm can be used to select wave components
Uses a [Fourier Transform](https://www.desmos.com/calculator/il9blockap) that I made a while ago on Desmos to try and reconstruct the original as closely as possible

## 🚀 Usage
<hr>

This project is still in development so alot of features may be missing still

Overall however, the usage may vary depending on your use case, for a general user:

1) Download the newest version of the analyser. ([Click here to access the newest build](https://github.com/Turtin/Signal_Analyser/latest))
Make sure to download the file called "Signal_Analyser.exe"

2) Place the file in a folder with your desired sound file(s), test files have also been provided

3) Open terminal, you can do this by pressing <kbd>Win</kbd> + <kbd>R</kbd> and then typing 
```
cmd
```
4) Make sure that you are in the same directory as the sound files and Signal_Analyser.exe
```commandline
cd "path/to/folder"
```
5) Run the file by simply typing out the name of the .exe file
```commandline
Signal_Analyser.exe
```
